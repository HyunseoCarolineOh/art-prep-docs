## P4. 문제 (1/2) — 시장 통증

**연 5만명 미대 수험생, 일평균 132분을 비정형 SNS 탐색에 허비**

- 학생 85% — 입시 평가 기준이 아닌 **개인 취향**으로 자료 선별
- 판단 오류율 **65%**
- 합격 데이터는 수도권 집중 → 지방 수험생 정보 격차 + 이동·거주비 이중 부담
- 학원비 월 200만원, 가계 부담 큼

---

## P5. 문제 (2/2) — 시장의 공백

**기존 채널은 모두 합격 전략 학습 도구가 아님**

| 채널 | 한계 |
|---|---|
| 미대입시닷컴 | 게시판 커뮤니티, 분석·큐레이션 없음 |
| 인스타·핀터레스트 | 취향 기반, 입시 기준과 무관 |
| 학원 | 비용·접근성 + 강사 편차 |
| 일반 AI 도구 | 미대 입시 채점 기준 미반영 |

→ **AI 분석 + 합격 DB 기반 전략 학습 도구가 시장에 전무**

## P6. 솔루션 (1/4) — Product Overview

**비전 AI 기반 5단계 실기 분석 리포트 + 합격 DB 1만건 교차분석 큐레이션, 10초 이내**

- 그림 업로드 → 비전 AI 분석 → 5단계 평가 + 개인화 큐레이션
- 모바일 우선, 학생이 자투리 시간에 사용
- 학원·강사 보조 도구로도 작동 (B2B2C 확장 가능)

---

## P7. 솔루션 (2/4) — 3축 모트

**신생팀이 복제 불가능한 3축 결합**

1. **6년 도메인 채점 기준의 코드화** — 분기형 프롬프트 트리
2. **합격 DB 라벨링 체계** — 합격작 1만건 + 입시 기준 라벨
3. **사용자 행동 편차 메트릭** — 저장·열람·필터 vs 합격 기준의 편차

→ 3축이 매칭 알고리즘에 직접 학습 신호로 들어감
→ 사용 누적 시 개인화 정확도 95%+ 수렴
→ 데이터·시간 비대칭이 1~2년 진입장벽 형성

---

## P8. 솔루션 (3/4) — 알고리즘 깊이

**비전 AI 모델 + 분기형 프롬프트 트리 + 합격 DB 라벨링**

- 입시 채점 항목(구도·명암·소재·완성도·표현력 등)을 분기 노드로 코드화
- 각 노드에서 합격 DB 라벨과 비교 → 5단계 평가 + 개선 포인트 산출
- 사용자 행동 데이터가 시간이 갈수록 모델 fine-tuning 신호로 누적
- 모델 선정은 검토 단계 (네이버 기술 리더 피드백 활용 기대)
