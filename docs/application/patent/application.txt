1. 발명의 설명

1.1 발명의 명칭

미대 입시 실기 자료의 AI 큐레이션 및 분석 시스템

(영문 명칭: AI-Based Curation and Analysis System for Art College Entrance Examination Practical-Skill Reference Materials)


1.2 기술분야

본 발명은 미술 대학 입시를 준비하는 수험생이 학습에 필요한 실기 참고 자료(이하 "실기 자료")를 효율적으로 탐색·평가·수집하도록 지원하는 시스템 및 방법에 관한 것이다. 보다 구체적으로는, 비전(Vision) 인공지능 모델과 합격생 작품 데이터베이스를 활용하여 사용자의 목표 대학·실기유형·현재 실력 수준에 부합하는 참고 자료를 자동으로 분석·평가하고, 사용자의 학습 행동 데이터를 기반으로 추천 정확도를 지속적으로 개선하는 개인화 큐레이션 시스템에 관한 것이다.


1.3 발명의 배경이 되는 기술

미대 입시 분야의 실기 시험은 대학별·전형별로 출제 경향, 채점 기준, 선호 화풍이 상이하다. 수험생은 자신의 목표 대학에 맞는 합격생 작품을 다수 분석하여 학습 방향을 설정해야 하지만, 종래에는 다음과 같은 자료 탐색 수단이 사용되어 왔다.

가. 일반 이미지 검색 및 SNS 플랫폼

수험생은 Pinterest, Instagram 등 비정형 이미지 공유 플랫폼에서 해시태그 또는 키워드 검색으로 참고 작품을 수집한다. 그러나 이러한 플랫폼은 전 세계 임의의 이미지가 혼재되어 있고, 대학·전형별 분류 체계가 부재하며, 입시 채점 기준을 반영한 평가 기능이 전무하다. 본 출원인의 사전 조사에 따르면 수험생 1인당 일평균 약 132분을 단순 탐색에 소비하면서도, 수집한 자료가 실제 목표 대학의 합격 기준과 부합하지 않는 비율이 약 65%에 달한다.

나. 입시 관련 커뮤니티 게시판

미대입시닷컴 등 커뮤니티 사이트는 수험생 간 정보 공유를 지원하나, 데이터가 비정형 게시글 형태로 누적되어 있고, 신뢰성 검증 절차가 없으며, 자동 분류·추천 기능이 부재하다.

다. 입시 미술 학원 (오프라인)

소수의 입시 미술 학원은 자체 보유한 합격작 아카이브를 학생에게 제공하나, 학원별·지역별 자료 편차가 심하고 월 200만원 이상의 수강료가 부담되어 경제적·지역적 진입장벽으로 작용한다. 합격 데이터가 수도권에 집중되어 지방 거주 수험생의 접근성이 현저히 낮다.

라. 범용 생성형 인공지능 도구

ChatGPT, Claude 등 범용 생성형 AI는 미대 입시 채점 기준을 학습하지 않았고, 대학별 전형의 평가 축(구도·명암·선질·완성도·표현력 등)을 분기 처리하지 못한다.

상기와 같이 종래기술은 (i) 탐색 시간이 과도하게 소요되고, (ii) 입시 기준을 반영하지 못한 채 개인 취향에 기반한 자료 수집이 이루어지며, (iii) 합격 데이터의 객관적 분석을 통한 전략적 학습이 불가능하고, (iv) 경제적·지역적 격차를 해소하지 못하는 문제점이 있다.


1.4 발명의 내용

1.4.1 해결하고자 하는 과제

본 발명은 상기 종래기술의 문제점을 해결하기 위한 것으로, 구체적인 과제는 다음과 같다.

첫째, 탐색 시간의 획기적 단축. 수험생이 실기 참고 자료를 발견·평가하는 데 소요되는 시간을 132분 수준에서 10초 이내로 단축한다.

둘째, 판단 오류율의 감소. 개인 취향에 기반한 비효율적 자료 선택을 배제하고, 다수의 합격작 데이터베이스에 대한 객관적 교차분석을 통해 입시 기준 부합도가 높은 자료만을 제시한다.

셋째, 목표 대학·전형별 맞춤 추천. 사용자의 목표 대학, 실기 유형(소묘·수채화·기초디자인 등), 현재 실력 수준, 선호 합격연도 범위를 입력받아, 해당 조건에 최적화된 합격작을 우선 노출한다.

넷째, 사용 데이터 누적에 따른 개인화 정확도 향상. 사용자의 저장(좋아요)·열람·필터 선택 등 행동 데이터를 실시간 수집·분석하여, 사용자의 잠재 취향 벡터와 합격 기준 벡터 간 편차를 보정하는 가중치를 추천 알고리즘에 반영함으로써, 사용 기간이 누적될수록 추천 정확도가 향상되도록 한다.

다섯째, 교육 기회의 균등화. 클라우드 기반 시스템으로 지역과 무관하게 동등한 접근을 제공하고, 오프라인 학원 대비 현저히 저렴한 비용으로 합격작 데이터에 접근 가능하도록 한다.


1.4.2 과제의 해결 수단

상기 과제를 해결하기 위하여, 본 발명에 따른 미대 입시 실기 자료의 AI 큐레이션 및 분석 시스템은 다음과 같은 구성요소를 포함한다.

가. 합격작 데이터베이스 구축부

미술 대학 합격생의 실기 작품 이미지 및 메타데이터를 저장하는 관리자 권한 전용 저장소를 포함한다. 메타데이터는 대학명(school), 실기 유형(exam_type), 합격 연도(pass_year), 지역(region), 기법·분위기 태그(tags) 및 비전 AI 분석 결과(ai_report, JSON 형식)를 포함한다.

나. 사용자 프로필 수집부

수험생으로부터 목표 대학, 목표 실기 유형, 현재 실력 수준, 선호 합격연도 범위를 입력받아 사용자 프로필을 생성하고 저장한다.

다. 비전 AI 분석부

상기 데이터베이스에 등록된 작품 이미지 또는 사용자가 업로드한 참고 이미지에 대하여, 비전 인공지능 모델에 (i) 이미지 URL, (ii) 대학·전형별 채점 기준이 코드화된 분기형 프롬프트, (iii) 사용자 프로필을 입력으로 제공하고, 다음 5단계의 분석을 수행하여 JSON 형식의 분석 리포트를 생성한다.

  ① 출제 문제 분석
  ② 평가 포인트 추출
  ③ 작품 의도 분석(구도·명암·선질·소재·완성도·표현력의 정량적 평가 포함)
  ④ 평가 기준 부합도 산출
  ⑤ 유사 합격작 안내

라. 메타데이터 매칭부

사용자 프로필과 합격작 데이터베이스의 메타데이터를 교차 비교하여 유사도를 계산한다. 유사도 산출은 코사인 유사도(Cosine Similarity) 또는 벡터 임베딩 기반 거리 계산 방식을 채택할 수 있다.

마. 추천 엔진부

상기 유사도 상위 N개(예: 10개)의 작품을 순위별로 선정하고, 각 작품에 대한 추천 근거 텍스트 및 유사 합격작 안내를 함께 제시한다. 초기 단계에서는 합격률 비중·최신 합격작 가중치·트렌드 반영 가중치를 포함하는 규칙 기반(rule-based) 가중치를 적용하고, 사용 데이터 누적 후에는 머신러닝 기반 추천으로 전환한다.

바. 행동 데이터 수집·보정부

사용자의 저장(좋아요), 반복 열람, 필터 선택 등의 행동을 실시간 스트리밍 방식으로 수집하여 행동 이력 테이블(saved_items, curation_sessions)에 기록하고, 사용자의 잠재 취향 벡터와 합격 기준 벡터 간 편차(gap)를 계산하여 추천 엔진부의 가중치를 지속적으로 보정한다.

사. 개인화 고도화부

규칙 기반 가중치 조정 단계에서 머신러닝 기반 학습 모델로 전환하여, 사용 기간이 증가함에 따라 사용자별 개인화 추천 정확도가 사전 설정 임계값(예: 95%) 이상으로 수렴하도록 한다.


1.4.3 발명의 효과

본 발명에 따르면, 종래기술 대비 다음과 같은 효과가 달성된다.

· 자료 탐색 시간: 종래 1인 평균 132분/일에서 본 발명 적용 시 10초 이내로 단축되어 약 99% 단축된다.
· 자료 판단 오류율: 종래 약 65%에서 본 발명 적용 시 약 15% 수준(목표)으로 감소되어 약 77% 감소된다.
· 개인화 추천 정확도: 종래기술에는 해당 기능이 없었으나, 본 발명은 사용 데이터 누적 시 95% 이상으로 수렴한다.
· 수험생 1인당 학습 비용: 종래 월 200만원 이상(학원)에서 본 발명 적용 시 월 29,900원(구독)으로 약 98% 절감된다.
· 자료 접근 지역 격차: 종래 수도권 집중이었으나, 본 발명은 클라우드 기반 전국 동등 접근을 제공하여 격차를 해소한다.

또한, 본 발명은 합격작 데이터베이스의 객관적 분석과 대학·전형별 분기형 프롬프트 트리를 결합함으로써, 6년 이상의 미대 입시 도메인 지식이 시스템에 코드화되어 있어 단순 모방으로는 복제가 곤란한 기술적 진입장벽을 형성한다.

【도 1】본 발명의 일 실시예에 따른 미대 입시 실기 자료의 AI 큐레이션 및 분석 시스템(100)의 전체 구성을 나타낸 블록도이다.

【도 2】본 발명의 일 실시예에 따른, 사용자 프로필 입력으로부터 추천 결과 출력 및 행동 데이터 피드백까지의 처리 과정을 나타낸 데이터 흐름도이다.

【도 3】본 발명의 일 실시예에 따른 비전 AI 분석부(130)가 수행하는 5단계 분석 ─ 출제 문제 분석, 평가 포인트 추출, 작품 의도 분석, 평가 기준 부합도 산출, 유사 합격작 안내 ─ 의 처리 순서를 나타낸 시퀀스 다이어그램이다.

【도 4】본 발명의 일 실시예에 따른 행동 데이터 수집·보정부(160)가 수행하는 가중치 보정 알고리즘으로서, 사용자 잠재 취향 벡터와 합격 기준 벡터 간의 편차를 산출하여 추천 엔진부(150)의 가중치를 보정하고, 누적 행동 수가 임계값을 초과하는 경우 개인화 고도화부(170)에 의한 머신러닝 기반 가중치 학습으로 전환되는 흐름을 나타낸 흐름도이다.


1.6 발명을 실시하기 위한 구체적인 내용

이하, 첨부된 도면을 참조하여 본 발명의 실시예를 상세히 설명한다.

가. 데이터베이스 스키마 예시

본 발명의 일 실시예에서, 합격작 데이터베이스(110)는 관계형 데이터베이스(예: PostgreSQL) 상에 다음과 같은 테이블로 구현될 수 있다.

  artworks (
    id              UUID PRIMARY KEY,
    image_url       TEXT NOT NULL,           -- 객체 저장소 내 이미지 경로
    school          TEXT NOT NULL,           -- 합격 대학명
    exam_type       TEXT NOT NULL,           -- 실기 유형
    region          TEXT,                    -- 지역
    pass_year       INTEGER NOT NULL,        -- 합격 연도
    tags            TEXT[],                  -- 기법·분위기 태그
    ai_report       JSONB,                   -- 비전 AI 분석 결과
    created_at      TIMESTAMPTZ DEFAULT now()
  )

  users (
    id              UUID PRIMARY KEY,
    target_school   TEXT,
    target_exam     TEXT,
    skill_level     INTEGER,                 -- 1~5
    pref_year_from  INTEGER,
    pref_year_to    INTEGER
  )

  saved_items (
    user_id         UUID REFERENCES users(id),
    artwork_id      UUID REFERENCES artworks(id),
    saved_at        TIMESTAMPTZ DEFAULT now(),
    PRIMARY KEY (user_id, artwork_id)
  )

  curation_sessions (
    id              UUID PRIMARY KEY,
    user_id         UUID REFERENCES users(id),
    filter_params   JSONB,                   -- 적용된 필터
    result_ids      UUID[],                  -- 추천 결과 작품 ID
    weight_vector   JSONB,                   -- 적용된 가중치 스냅샷
    created_at      TIMESTAMPTZ DEFAULT now()
  )


나. 비전 AI 분석부의 실시예

비전 AI 분석부(130)는 외부 비전 인공지능 API(예: 멀티모달 LLM)를 호출하여 5단계 분석을 수행한다. 입력 프롬프트는 다음과 같이 사용자 프로필 및 대학·전형별 채점 기준에 따라 분기되어 구성된다.

  [시스템 프롬프트 — 대학별 분기]
  "당신은 {target_school}의 {target_exam} 전형 채점 평가관이다.
   본 대학의 평가 축은 [구도 30%, 명암 25%, 선질 20%, 완성도 15%, 표현력 10%]이며,
   합격선 기준은 다음과 같다: {school_passing_criteria_json}."

  [사용자 메시지]
  "다음 작품 이미지에 대하여 5단계 분석을 수행하고 결과를 JSON으로 반환하라.

분석 결과 ai_report(JSONB)는 다음과 같은 구조를 가질 수 있다.

다. 추천 엔진의 실시예

추천 엔진부(150)는 다음과 같은 가중 점수 함수를 통해 작품의 추천 순위를 결정한다.

  Score(artwork) = w1 · MetadataMatch(user, artwork)
                + w2 · CriteriaMatchRate(artwork.ai_report)
                + w3 · TrendBoost(artwork.pass_year)
                + w4 · PersonalizationDelta(user_behavior_vector, artwork)

여기서 w1 내지 w4는 행동 데이터 수집·보정부(160)에 의해 사용자별로 실시간 조정되는 가중치이다. 초기에는 (w1=0.4, w2=0.3, w3=0.2, w4=0.1) 등 사전 정의 값으로 시작하고, 사용 데이터가 누적됨에 따라 머신러닝 기반 회귀 모델로 갱신된다.


라. 행동 데이터 보정의 실시예

행동 데이터 수집·보정부(160)는 사용자가 저장(좋아요)·열람·필터 선택을 수행할 때마다 saved_items 및 curation_sessions 테이블에 기록을 적재하고, 일정 시점마다 다음과 같은 편차 계산을 수행한다.

  preference_vector(user) = aggregate(saved_items, curation_sessions)
  gap(user) = preference_vector(user) − passing_criteria_vector(target_school)
  adjusted_weights(user) = base_weights · normalize(gap)

산출된 adjusted_weights는 다음 추천 호출 시 즉시 반영되어, 사용자의 잠재 취향과 합격 기준 간의 거리를 점진적으로 좁히는 방향으로 추천이 보정된다.


마. 추가 실시예 — B2B 학원 대시보드

본 발명의 다른 실시예에 따르면, 합격작 데이터베이스 및 행동 데이터를 집계하여 입시 미술 학원 운영자에게 학원별 작품 분포, 학원별 인기 작품, 학원 가입자 추이 등을 제공하는 B2B 대시보드 기능을 추가로 포함할 수 있다. 본 실시예는 일별 배치(예: KST 06:00)로 갱신되는 구체화 뷰(Materialized View)에 기반하여 구현될 수 있다.
