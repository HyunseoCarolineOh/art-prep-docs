## 1. 프로젝트 개요

### 1-1. 목적
- **모두의 창업 프로젝트 2026** 신청서 아이디어 설명 이미지 (선택 제출 → 실질 필수)
- 심사위원(5~7명 추정)이 300개 이상 신청서 중 3분 내 핵심 메시지를 캐치하도록 설계
- 텍스트 기반 신청서의 한계를 시각 자료로 보완, 설득력 극대화

### 1-2. 제약사항
- **수량**: 5장 고정 (1장 부족/초과 모두 불가)
- **해상도**: 최소 1920×1080px (가로형 권장, 심사위원 PC 환경)
- **파일명**: 영문 (예: `modoo-artprep-image-1.png`)
- **제출 방식**: 모두의 창업 웹사이트 업로드 (파일 크기 제한 미확인, 5MB 이내 권장)
- **메시지 원칙**: 1장당 1개 핵심 메시지 (중복 금지)

### 1-3. 브랜드 톤 (아트프렙)
- **키워드**: 전문성(6년 도메인 경험), 데이터 기반(합격 DB·통계), 접근성(모바일 최적화·직관적 UI)
- **컬러 방향**: 신뢰(블루·네이비) + 에너지(오렌지·옐로우 액센트) — 에듀테크 일반론 참고 OK
- **폰트**: 고딕 계열 (가독성 최우선), 수치·통계는 숫자 강조 (굵기 차이로 위계)
- **스타일**: 인포그래픽 중심 (깔끔한 아이콘·그래프·플로우), 사진은 최소화 (스톡 이미지 사용 시 자연스러움 필수)

## 2. 이미지별 구성안 (5장)

### 이미지 1 — 문제 상황 (Problem)

**제목**: 연 5만명 수험생, 132분 탐색 허비 + 판단 오류율 65%

**비주얼 구성**:
- **좌측 (50%)**: 학생이 스마트폰으로 인스타그램·핀터레스트를 넘기는 모습  
  → 스톡 이미지 또는 실루엣 일러스트 (저작권 주의)  
  → 스크롤 화면에 실제 입시 자료 이미지 레이어 (현실감 ↑)
- **우측 (50%)**: 통계 그래픽  
  - **132분** (크게 강조, 예: 72pt 볼드)  
  - → 하단 서브텍스트: "일평균 SNS 탐색 시간" (24pt 레귤러)  
  - **65%** (크게 강조, 예: 72pt 볼드, 빨간색 액센트)  
  - → 하단 서브텍스트: "판단 오류율 (취향 기반 선택)" (24pt 레귤러)
- **하단 캡션**: "입시 기준 아닌 취향으로 선택 → 방향 정체 반복" (18pt, 중앙 정렬)

**핵심 메시지**: 구조적 문제 + 도구 부재가 만든 시간·비용 낭비

**참고 레이아웃**: 좌우 2단 레이아웃 (50:50), 배경은 화이트 또는 라이트 그레이

### 이미지 2 — AI 솔루션 (Solution)

**제목**: 비전 AI 분석 + 합격 DB 1만건 교차분석 → 10초 큐레이션

**비주얼 구성**:
- **상단 (30%)**: 사용자가 그림 업로드하는 모바일 UI 스크린샷  
  → 현재 프로토타입 없으므로, 와이어프레임 목업 또는 피그마 스타일 UI 목업 사용  
  → 버튼: "AI 분석 시작" 액센트 컬러로 강조
- **중앙 (40%)**: 비전 AI 분석 → 5단계 평가 프로세스 (아이콘 플로우)  
  1. 업로드 (아이콘: 이미지 업로드)  
  2. AI 분석 (아이콘: 로봇·뇌 모양)  
  3. 채점 기준 매칭 (아이콘: 체크리스트)  
  4. 합격 DB 교차분석 (아이콘: 데이터베이스)  
  5. 큐레이션 결과 (아이콘: 별·트로피)  
  → 화살표로 플로우 연결, 각 단계 1줄 설명 (12pt)
- **하단 (30%)**: 합격 DB 교차분석 → Top10 큐레이션 결과 (이미지 그리드)  
  → 3×3 그리드에 미대입시 작품 썸네일 (실제 이미지 없으면 placeholder 색상 블록)  
  → 상단 라벨: "당신에게 추천하는 합격 작품 Top 10"

**핵심 메시지**: 132분 → 10초 단축, 합격 기준 기반 객관적 추천

**참고 레이아웃**: 3단 세로 레이아웃 (30:40:30), 플로우 중앙 강조

**비주얼 구성**:

  - 서브텍스트: "6년 현장 경험 → AI 프롬프트 트리 변환"  
  - 배지: "복제 불가 1"

  **2. 합격 DB 라벨링 체계** (중)  
  - 아이콘: 데이터베이스 + 태그  
  - 서브텍스트: "1만건 합격작 + 입시 기준 라벨링"  
  - 배지: "복제 불가 2"

  **3. 사용자 행동 편차 메트릭** (우)

  - 서브텍스트: "저장·열람 vs 합격 기준 간 편차 측정"  
  - 배지: "복제 불가 3"

- **하단 캡션**: "3축이 매칭 알고리즘에 직접 학습 신호 → 사용 누적 시 개인화 정확도 95%+ 수렴" (14pt, 중앙 정렬, 배경 색상 블록)

**핵심 메시지**: 신생팀 복제 불가, 데이터·시간 비대칭이 1~2년 진입장벽 형성

**참고 레이아웃**: 3단 가로 레이아웃 (33:33:33), 각 축 아이콘 위주

## 3. 디자인 가이드라인

- **Accent**: 오렌지 (#F97316) 또는 옐로우 (#FBBF24) — 에너지·강조

- **Text**: 다크 그레이 (#1F2937) — 본문, 블랙 (#000000) — 헤딩
- **Error/Alert**: 레드 (#EF4444) — 판단 오류율 65% 같은 부정 지표

### 3-2. 타이포그래피
- **제목 (각 이미지 상단)**: 48~60pt, 굵게 (Bold)
- **수치 (132분, 65%, 22%)**: 60~72pt, 초굵게 (Extra Bold), 액센트 컬러
- **본문**: 18~24pt, 레귤러 (Regular)
- **캡션**: 12~14pt, 라이트 (Light) 또는 레귤러
- **폰트 추천**: Pretendard (오픈소스), Noto Sans KR, Inter (영문 수치)

### 3-3. 아이콘 스타일
- **스타일**: 라인 아이콘 (Outline) 또는 플랫 (Flat) — 일관성 유지
- **크기**: 64~96px (아이콘 단독), 48px (플로우 내 아이콘)

### 3-4. 레이아웃 원칙
- **여백**: 충분한 패딩 (최소 80px 상하좌우) — 답답함 제거
- **위계**: 제목 → 수치 → 서브텍스트 순으로 크기·색상 차별화
- **정렬**: 좌우 대칭 또는 그리드 기반 (시각적 안정감)
- **배경**: 단색 또는 그라데이션 (패턴·텍스처 자제, 가독성 우선)
