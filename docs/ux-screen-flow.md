# UX 화면 플로우 명세 — ArtPrep MVP

**최종 업데이트**: 2026-03-11
**담당**: 오현서 (CEO)

---

## 전체 화면 플로우

```
앱 시작
  ├── [신규 사용자] → 로그인/회원가입 화면
  │       ↓ 회원가입 완료
  │   온보딩 목표설정 화면 (최초 1회)
  │       ↓ 목표 저장
  └── [기존 사용자] → 자동 로그인
          ↓
    [메인 탭 네비게이션]
      ├── 홈 탭 (큐레이션)
      │     └── 작품 카드 탭 → 작품 상세 화면
      ├── 저장함 탭
      │     └── 저장 작품 탭 → 작품 상세 화면
      └── 설정 탭
            └── 목표 재설정 → 온보딩 목표설정 화면
```

## 화면 1: 로그인 (`app/(auth)/login.tsx`)

### 목적
기존 사용자 로그인

### UI 구성
```
[로고 + 서비스명]

[이메일 입력 필드]
[비밀번호 입력 필드]

[로그인 버튼]  ← 주요 CTA

[회원가입 링크]
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 로그인 버튼 탭 | Supabase Auth 이메일 로그인 → 성공 시 홈 이동 |
| 로그인 실패 | 에러 메시지 표시 ("이메일 또는 비밀번호를 확인해주세요") |
| 회원가입 링크 탭 | 회원가입 화면으로 이동 |
| 앱 재시작 | 세션 유지 → 자동 로그인 (홈으로 바로 이동) |

### 검증 기준
- [ ] 잘못된 이메일 형식 입력 시 즉시 에러 표시
- [ ] 로그인 성공 시 홈 탭으로 이동
- [ ] Supabase Dashboard에서 last_sign_in 확인 가능

## 화면 2: 회원가입 (`app/(auth)/signup.tsx`)

### 목적
신규 사용자 계정 생성

### UI 구성
```
[로고]

[이메일 입력 필드]
[비밀번호 입력 필드] (최소 8자)
[비밀번호 확인 필드]

[회원가입 버튼]

[로그인 링크]
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 회원가입 버튼 탭 | Supabase Auth 회원가입 → 성공 시 목표설정 화면으로 이동 |
| 이미 가입된 이메일 | "이미 사용 중인 이메일입니다" 에러 |
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다" 에러 |

### 검증 기준
- [ ] 회원가입 성공 → Supabase Auth 대시보드에서 사용자 생성 확인
- [ ] 온보딩 화면으로 자동 이동

## 화면 3: 온보딩/목표설정 (`app/(onboarding)/goals.tsx`)

### 목적
사용자의 목표 대학·전형·실기 유형 설정 (큐레이션 기반)

### UI 구성 (3단계 스텝)
```
[진행 단계 표시] ①②③

--- 1단계: 목표 대학 선택 ---
[대학명 목록 — 체크박스 또는 버튼 선택]
홍익대학교 / 서울대학교 / 이화여자대학교 /
국민대학교 / 건국대학교 / 경희대학교 / ...

[다음 버튼]

--- 2단계: 전형 선택 ---
[수시] [정시]

[다음 버튼]

--- 3단계: 실기 유형 선택 ---
[소묘] [수채화] [기초디자인]
[발상과표현] [사고의전환] [색채구성] [기타]

[완료 버튼]
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 완료 버튼 탭 | user_goals 테이블에 저장 → 홈 탭으로 이동 |
| 뒤로가기 | 이전 단계로 이동 |
| 설정에서 재진입 | 기존 값 로드하여 수정 가능 |

### 검증 기준
- [ ] 선택값이 Supabase `user_goals` 테이블에 저장
- [ ] 완료 후 홈 화면에서 해당 목표 기반 작품 표시

## 화면 4: 홈 (큐레이션) (`app/(tabs)/index.tsx`)

### 목적
목표 기반 큐레이션된 작품 리스트 탐색

### UI 구성
```
[헤더] "내 큐레이션" + 목표 표시 (홍익대 수시 기초디자인)

[필터 칩 스크롤]
[전체] [기초디자인] [소묘] [서울] [경기인천] ...

[작품 카드 그리드 — 2열]
┌──────┐ ┌──────┐
│ 이미지│ │ 이미지│
│ 대학명│ │ 대학명│
│ 유형  │ │ 유형  │
│ ♡저장│ │ ♡저장│
└──────┘ └──────┘
(무한스크롤)

[로딩 인디케이터] ← 다음 페이지 로드 시
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 작품 카드 탭 | 작품 상세 화면으로 이동 |
| ♡ 버튼 탭 | 저장함에 추가/제거 (즉시 반영) |
| 필터 칩 탭 | 해당 유형/지역으로 필터링 |
| 스크롤 끝 도달 | 다음 20건 자동 로드 (무한스크롤) |
| 새로고침 (당겨서) | 최신 데이터 다시 로드 |

### 페이지네이션
- 1회 로드: 20건
- 정렬: success_score 내림차순
- 필터: artwork_type + 선택한 필터 칩

### 검증 기준
- [ ] 목표설정과 다른 유형 작품이 필터됨
- [ ] 무한스크롤 시 추가 작품 로드됨
- [ ] 저장 버튼 즉시 반영

## 화면 5: 작품 상세 (`app/artwork/[id].tsx`)

### 목적
작품 이미지 + GPT-4 5단계 분석 리포트 조회

### UI 구성
```
[뒤로가기] [저장 ♡]

[작품 이미지 — 전체 너비]

[작품 정보]
대학명 | 전형 | 실기유형 | 연도

━━━━━━━━━━━━━━━━━
[AI 분석 리포트]

● 1단계: 출제의도
[텍스트]

● 2단계: 학교 성향
[텍스트]

● 3단계: 조형요소
  구도    ████████░░  85점
  형태    ███████░░░  78점
  색채    █████████░  90점
  표현력  ████████░░  82점

● 4단계: 평가기준 부합도
[텍스트]

● 5단계: 종합 리포트
  종합 점수: 84점
  핵심 강점: ...
  핵심 약점: ...
  개선 방향: ...
━━━━━━━━━━━━━━━━━
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 화면 진입 | 분석 리포트 DB 조회 → 없으면 Edge Function 호출 |
| 분석 로딩 중 | "AI가 분석 중입니다..." + 스켈레톤 UI |
| 저장 버튼 탭 | saved_artworks 추가/제거 |
| 이미지 탭 | 이미지 전체화면 보기 |

### 로딩 상태 처리
1. 분석 리포트 있음 → 즉시 표시
2. 분석 리포트 없음 → Edge Function 호출 (5~15초) → 완료 후 표시
3. 오류 발생 → "분석을 불러올 수 없습니다. 다시 시도해주세요."

### 검증 기준
- [ ] 분석 리포트가 올바른 5단계 구조로 표시
- [ ] 조형요소 점수가 프로그레스 바로 시각화
- [ ] 재방문 시 캐시된 리포트 즉시 로드 (재분석 없음)

## 화면 6: 저장함 (`app/(tabs)/saved.tsx`)

### 목적
사용자가 저장한 작품 목록 조회

### UI 구성
```
[헤더] "저장함"

[작품 카드 리스트 — 2열]
(홈 화면과 동일한 카드 컴포넌트)

[저장 작품 없을 때]
"저장한 작품이 없습니다.
홈에서 마음에 드는 작품을 저장해보세요."
[홈으로 버튼]
```

### 동작 명세
| 액션 | 동작 |
|------|------|
| 작품 카드 탭 | 작품 상세 화면으로 이동 |
| ♡ 버튼 탭 (해제) | 저장 취소 → 리스트에서 제거 |

### 검증 기준
- [ ] 홈에서 저장한 작품이 이 화면에 표시
- [ ] 저장 해제 시 즉시 목록에서 사라짐


| 로그아웃 버튼 탭 | Supabase Auth 로그아웃 → 로그인 화면으로 이동 |

## 공통 컴포넌트

### ArtworkCard
- 작품 썸네일 이미지
- 대학명, 실기 유형 텍스트
- 저장 버튼 (♡)
- 탭 시 상세 화면으로 이동

### FilterChip
- 텍스트 칩 버튼
- 선택 시 강조 스타일
- 복수 선택 가능

### AnalysisReport
- 5단계 섹션 렌더링
- 조형요소 프로그레스 바
- 로딩 스켈레톤 상태

## 네비게이션 구조

```
RootLayout (_layout.tsx)
  ├── 인증 미완료 → AuthStack
  │     ├── login
  │     └── signup
  ├── 목표 미설정 → OnboardingStack
  │     └── goals
  └── 인증 완료 + 목표 설정 완료 → TabsLayout
        ├── (tabs)/index     ← 홈
        ├── (tabs)/saved     ← 저장함
        ├── (tabs)/settings  ← 설정
        └── artwork/[id]     ← 상세 (탭 위에 스택)
```
