# UX 화면 흐름 — ArtPrep MVP

**최종 업데이트**: 2026-03-14

---

## 1. 전체 화면 목록 및 라우트

| 화면 | 라우트 | 파일 |
|------|--------|------|
| 로그인 | `/login` | `app/(auth)/login/page.tsx` |
| 회원가입 | `/signup` | `app/(auth)/signup/page.tsx` |
| 온보딩 — 목표 설정 | `/onboarding/goals` | `app/onboarding/goals/page.tsx` |
| 홈 — 큐레이션 피드 | `/` | `app/page.tsx` |
| 작품 상세 | `/artwork/[id]` | `app/artwork/[id]/page.tsx` |
| 저장함 | `/saved` | `app/saved/page.tsx` |
| 설정 | `/settings` | `app/settings/page.tsx` |

---

## 2. 네비게이션 흐름도

```
[미인증 사용자]
  → 보호된 라우트 접근 시 /login?returnTo=원래경로 리다이렉트

[로그인 / 소셜 가입 성공]
  → user_goals 없음 (신규):  /onboarding/goals
  → user_goals 있음 (기존):  / (홈) 또는 returnTo 경로

[온보딩 완료]
  → / (홈)

[홈 탭 네비게이션]
  홈(/) ↔ 저장함(/saved) ↔ 설정(/settings)

[홈/저장함 작품 카드 클릭]
  → /artwork/[id]

[설정 → 목표 재설정]
  → /onboarding/goals (완료 후 /settings 복귀)

[로그아웃]
  → /login
```

**인증 미들웨어**: Next.js middleware에서 `/`, `/saved`, `/settings`, `/artwork/*` 접근 시 세션 확인 → 미인증이면 `/login?returnTo=...` 리다이렉트

### 화면 1: 로그인 (`/login`)

**목적**: 기존 회원 로그인 (이메일, Google, Kakao)

```
┌─────────────────────────────┐
│           ArtPrep           │
│                             │
│  이메일   [_______________] │
│  비밀번호 [_______________] │
│  ⚠ 이메일 또는 비밀번호가   │
│    올바르지 않습니다. (에러) │
│                             │
│     [      로그인      ]    │
│                             │
│  ────── 또는 ──────────     │
│  [G  Google로 계속하기  ]   │
│  [K  Kakao로 계속하기   ]   │
│                             │
│  계정이 없으신가요? [회원가입]│
└─────────────────────────────┘
```

**상태**

| 상태 | 처리 |
|------|------|
| 로딩 중 | 로그인 버튼 비활성화 + 스피너 |
| 이메일 없음 / 비밀번호 불일치 | 인라인 에러 메시지 표시 |
| 성공 | returnTo 경로 또는 `/` 이동 |

**인터랙션**
- 이메일 + 비밀번호 제출 → `supabase.auth.signInWithPassword()`
- Google 버튼 → `supabase.auth.signInWithOAuth({ provider: 'google' })`
- Kakao 버튼 → `supabase.auth.signInWithOAuth({ provider: 'kakao' })`
- [회원가입] 링크 → `/signup`

### 화면 2: 회원가입 (`/signup`)

**목적**: 신규 회원 계정 생성

```
┌─────────────────────────────┐
│           회원가입           │
│                             │
│  이메일   [_______________] │
│  비밀번호 [_______________] │
│  비밀번호 확인 [___________]│
│                             │
│     [      가입하기      ]  │
│                             │
│  ────── 또는 ──────────     │
│  [G  Google로 계속하기  ]   │
│  [K  Kakao로 계속하기   ]   │
│                             │
│  이미 계정이 있으신가요? [로그인] │
└─────────────────────────────┘
```

**상태**

| 상태 | 처리 |
|------|------|
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다." 인라인 에러 |
| 이미 가입된 이메일 | "이미 사용 중인 이메일입니다." 인라인 에러 |
| 성공 | `/onboarding/goals` 이동 |

**인터랙션**
- 이메일 + 비밀번호 제출 → `supabase.auth.signUp()`
- 소셜 가입: 로그인과 동일 OAuth 흐름 (신규면 자동 가입)

### 화면 3: 온보딩 — 목표 설정 (`/onboarding/goals`)

**목적**: 큐레이션 기준이 되는 목표 대학 / 전형 / 실기유형 설정

```
STEP 1/3  ●○○
────────────────────────────────
목표 대학을 선택해주세요

[홍익대] [서울대] [이화여대]
[국민대] [건국대] [중앙대]
[성신여대] [세종대] [경희대]
[한국예술종합학교] ...

                      [다음 →]

────────────────────────────────
STEP 2/3  ●●○
────────────────────────────────
전형을 선택해주세요

[       수시       ]  [       정시       ]

          [← 이전]  [다음 →]

────────────────────────────────
STEP 3/3  ●●●
────────────────────────────────
실기 유형을 선택해주세요 (복수 선택)

[소묘 ✔] [수채화] [유화]
[기초디자인 ✔] [발상과표현] [사고의전환]
[정밀묘사] [칸만화] ...

          [← 이전]  [완료 →]
```

**상태**

| 상태 | 처리 |
|------|------|
| 선택 없이 [다음] 클릭 | "선택해주세요" 토스트 메시지 |
| STEP 3 실기유형 미선택 | [완료] 버튼 비활성화 |
| 완료 | `user_goals` UPSERT → `/` 이동 |

**인터랙션**
- 각 단계 선택 후 [다음] → 다음 스텝 표시
- [이전] → 이전 스텝으로 이동 (선택값 유지)
- [완료] → `supabase.from('user_goals').upsert()` → 홈 이동
- 설정에서 재진입 시 기존 값 pre-select 후 동일 흐름 진행

**데이터**: `user_goals` 테이블 UPSERT

### 화면 4: 홈 — 큐레이션 피드 (`/`)

**목적**: 사용자 목표 기반 합격 실기자료 탐색

```
┌─────────────────────────────────┐
│ [홍익대 ✔] [수시 ✔] [기초디자인 ✔] │
│ [연도: 전체 ▼]  [지역: 전체 ▼]   │
├─────────────────────────────────┤
│ ┌──────────┐  ┌──────────┐     │
│ │  이미지   │  │  이미지   │     │
│ │          │  │          │     │
│ └──────────┘  └──────────┘     │
│ 홍익대 2024   서울대 2023       │
│                                 │
│ ┌──────────┐  ┌──────────┐     │
│ │  이미지   │  │  이미지   │     │
│ │          │  │          │     │
│ └──────────┘  └──────────┘     │
│ 이화여대 2024  국민대 2023      │
│                                 │
│          ↓ 무한 스크롤          │
├─────────────────────────────────┤
│  🏠 홈    ♡ 저장함    ⚙ 설정  │
└─────────────────────────────────┘
```

**빈 상태**
```
조건에 맞는 작품이 없습니다.
현재 필터: 홍익대 | 수시 | 기초디자인

[다른 대학 보기]
```

**상태**

| 상태 | 처리 |
|------|------|
| 로딩 중 | 카드 스켈레톤 (4개) 표시 |
| 데이터 있음 | 2열 그리드 표시 |
| 빈 상태 | 빈 상태 메시지 + [다른 대학 보기] 버튼 |
| 스크롤 끝 | 추가 로딩 스피너 → 다음 20개 로드 |

**인터랙션**
- 필터 칩 클릭 → 해당 필터 토글 (목표 칩은 토글 불가, 표시만)
- 연도/지역 드롭다운 → 선택 즉시 리페치
- 작품 카드 클릭 → `/artwork/[id]`
- [다른 대학 보기] → 대학 필터 제거 후 재조회

**데이터**
```typescript
supabase.from('artworks')
  .select('*, analysis_reports(score)')
  .eq('artwork_type', userGoal.artwork_type)
  .eq('university', userGoal.target_university)
  .eq('exam_type', userGoal.exam_type)
  .order('success_score', { ascending: false })
  .range(from, to)   // 20개 단위
```

### 화면 5: 작품 상세 (`/artwork/[id]`)

**목적**: 작품 이미지 + Gemini 5단계 분석 리포트 조회

```
┌─────────────────────────────────┐
│ [← 뒤로]              [♡ 저장] │
├─────────────────────────────────┤
│                                 │
│         작품 이미지 전체너비      │
│                                 │
├─────────────────────────────────┤
│ 홍익대  |  2024  |  수시  |  기초디자인 │
├──── AI 분석 ────────────────────┤
│                                 │
│ [미분석]                        │
│  ┌───────────────────────────┐  │
│  │  🔍 AI 5단계 분석 요청하기 │  │
│  └───────────────────────────┘  │
│  이 작품은 아직 분석되지 않았습니다. │
│                                 │
│ [로딩]                          │
│  ⏳ AI가 분석 중입니다... (3~5초) │
│                                 │
│ [완료]                          │
│  1단계: 출제의도 ─────────────  │
│  [분석 텍스트]                  │
│                                 │
│  2단계: 학교 성향 ────────────  │
│  [분석 텍스트]                  │
│                                 │
│  3단계: 조형요소 ─────────────  │
│  구도  ████████░░ 85점          │
│  형태  ███████░░░ 78점          │
│  색채  █████████░ 90점          │
│  표현  ████████░░ 82점          │
│                                 │
│  4단계: 평가기준 부합도 ────────  │
│  [분석 텍스트]                  │
│                                 │
│  5단계: ★ 84점 ───────────────  │
│  강점: [요약]                   │
│  개선: [방향]                   │
└─────────────────────────────────┘
```

**AI 분석 상태 전이**
```
미분석 → [버튼 클릭] → 로딩(3~5초) → 분석 완료
이미 분석됨 → 바로 완료 화면 표시
```

**인터랙션**
- [AI 5단계 분석 요청하기] 버튼 클릭 → Edge Function 호출
  - `supabase.functions.invoke('analyze-artwork', { body: { artwork_id } })`
  - 응답 후 화면 새로고침 (리포트 표시)
- [♡ 저장] / [♥ 저장됨] 토글
  - 저장: `supabase.from('saved_artworks').insert()`
  - 해제: `supabase.from('saved_artworks').delete().match()`
- [← 뒤로] → 이전 페이지

**데이터**
- `artworks` 단건 조회
- `analysis_reports` 조회 (있으면 바로 표시)
- `saved_artworks` 해당 유저 저장 여부 확인

## 5. 에러 / 엣지 케이스

| 상황 | 처리 |
|------|------|
| 네트워크 오류 | 토스트 "잠시 후 다시 시도해주세요" |
| AI 분석 실패 (Edge Function 500) | "분석에 실패했습니다. 다시 시도해주세요." + 재시도 버튼 |
| 작품 없음 (404) | "존재하지 않는 작품입니다." + [홈으로] 버튼 |
| 세션 만료 | 자동 `/login?returnTo=...` 리다이렉트 |
| 홈 빈 상태 | "조건에 맞는 작품이 없습니다." + [다른 대학 보기] 버튼 |
| 저장함 빈 상태 | "저장한 작품이 없습니다." + [작품 탐색하기] 버튼 |
