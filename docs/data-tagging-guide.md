# 데이터 태깅 가이드 — 작품 메타데이터 입력 기준

**최종 업데이트**: 2026-03-11
**담당**: 오현서 (CEO)
**목표**: 1,000건 → 1만건 정형화

---

## 개요

ArtPrep 큐레이션 엔진의 정확도는 데이터 품질에 달려 있습니다.
이 가이드는 실기자료 이미지를 Supabase DB에 입력할 때 따라야 할 기준입니다.

---

## 1. 필드별 입력 기준

### image_url (필수)
- Supabase Storage `artworks` 버킷에 업로드 후 URL 입력
- 파일명 규칙: `{대학코드}_{전형}_{유형}_{연도}_{순번}.jpg`
  - 예: `hongik_susi_kicho_2024_001.jpg`
- 이미지 크기: 최소 800x600px, 최대 4000x3000px
- 포맷: JPG 또는 PNG

### university (필수)
정확한 대학명을 아래 목록에서 선택:

| 코드 | 대학명 |
|------|--------|
| seoul_nat | 서울대학교 |
| hongik | 홍익대학교 |
| ewha | 이화여자대학교 |
| kookmin | 국민대학교 |
| konkuk | 건국대학교 |
| kyunghee | 경희대학교 |
| sungshin | 성신여자대학교 |
| duksung | 덕성여자대학교 |
| sejong | 세종대학교 |
| inha | 인하대학교 |
| ajou | 아주대학교 |
| chungnam | 충남대학교 |
| jeonbuk | 전북대학교 |
| yeungnam | 영남대학교 |
| other | 기타 |

> 목록에 없는 대학은 `other`로 입력 후 `tags`에 실제 대학명 추가

### region (필수)
| 값 | 기준 |
|----|------|
| `서울` | 서울 소재 대학 |
| `경기인천` | 경기, 인천 소재 대학 |
| `지방` | 그 외 지방 소재 대학 |

### exam_type (필수)
| 값 | 설명 |
|----|------|
| `수시` | 학생부종합, 학생부교과 등 수시전형 |
| `정시` | 대학수학능력시험 기준 정시전형 |

### artwork_type (필수)
| 값 | 설명 |
|----|------|
| `소묘` | 연필 소묘, 목탄 소묘 |
| `수채화` | 수채 채색 |
| `유화` | 유화 채색 |
| `기초디자인` | 기초디자인 (평면 구성 포함) |
| `발상과표현` | 발상과 표현 |
| `사고의전환` | 사고의 전환 |
| `정밀묘사` | 정밀묘사 |
| `색채구성` | 색채 구성 |
| `입체조형` | 입체 조형물 |
| `기타` | 위 분류에 해당하지 않는 경우 |

### year (권장)
- 작품 제작 연도 (4자리 숫자)
- 불명확한 경우 `null` 입력
- 입시 연도 기준 (예: 2024년 입시에 사용된 자료 → 2024)

### tags (선택, 배열)
검색 및 필터링을 위한 추가 태그:

```
// 표현 기법 태그
"극사실", "반추상", "평면적", "입체적"

// 주제/소재 태그
"인물", "정물", "풍경", "상상화", "제시물"

// 합격 정보 태그 (알 수 있는 경우)
"합격작", "최우수작", "장학생작"

// 특수 상황 태그
"제한시간3시간", "A4용지", "큰화면"
```

### success_score (필수, 0~100)
합격 기여도 점수. 아래 기준 참고:

| 점수 범위 | 기준 |
|----------|------|
| 90~100 | 최우수 합격작 (입시 우수 사례) |
| 80~89 | 합격작 (일반 합격 수준) |
| 70~79 | 합격권 (합격 가능성 높음) |
| 60~69 | 준합격 (추가합격 수준) |
| 50~59 | 불합격권 (참고용 자료) |
| 0~49 | 실패 사례 (개선점 파악용) |

> 합격 여부가 명확하지 않은 경우: 전문가 판단에 의해 50~70 부여

## 2. 입력 방법

### Supabase 대시보드 직접 입력 (소량 — 10건 이하)
1. Supabase Dashboard → Table Editor → `artworks`
2. `Insert row` 클릭
3. 각 필드 직접 입력

### CSV 일괄 업로드 (대량 — 100건 이상)
1. 아래 CSV 템플릿 다운로드
2. 내용 채우기
3. Supabase Dashboard → Table Editor → `Import data`

**CSV 템플릿**:
```csv
image_url,university,region,exam_type,artwork_type,year,tags,success_score
https://...supabase.co/storage/v1/object/public/artworks/파일명.jpg,홍익대학교,서울,수시,기초디자인,2024,"{합격작,기초디자인}",85
```

---

## 3. 이미지 업로드 순서

1. **이미지 정리**: 파일명을 규칙에 맞게 변경
2. **Supabase Storage 업로드**:
   - Supabase Dashboard → Storage → `artworks` 버킷
   - 파일 드래그 앤 드롭 업로드
3. **URL 복사**: 업로드된 파일 클릭 → URL 복사
4. **DB 입력**: 복사한 URL을 `image_url`에 입력

---

## 4. 데이터 품질 체크리스트

입력 완료 후 아래 기준으로 검토:

- [ ] 이미지가 실제로 로드되는가 (URL 확인)
- [ ] 대학명이 정확한가
- [ ] 실기 유형이 올바른가 (소묘인데 채색으로 입력하지 않았는가)
- [ ] success_score가 근거 있는 값인가
- [ ] 중복 작품이 없는가 (동일 이미지 2회 입력 금지)

## 5. 1,000건 → 1만건 로드맵

| 단계 | 수량 | 시기 | 방법 |
|------|------|------|------|
| 현재 | ~1,000건 | 2026년 3월 | 수동 수집 완료 |
| Phase 1 | 2,000건 | 2026년 4월 | 태깅 기준으로 정형화 시작 |
| Phase 2 | 5,000건 | 2026년 6월 | 파일럿 학원 협력 데이터 추가 |
| Phase 3 | 1만건 | 2026년 12월 | B2B 파트너 데이터 + 자동 수집 |

### 블루닷 학원 협력 데이터
- 블루닷 입시미술학원 제공 데이터: 연 500~1,000건 예상
- 입력 기준: 이 가이드 공유 후 협력 강사가 직접 입력 가능
- 검증: 오현서 CEO가 최종 검토 후 승인

## 6. 주의사항

- **저작권**: 합격생 동의 없이 개인 식별 정보 포함 금지
- **이미지 품질**: 흐릿하거나 일부 잘린 이미지는 제외
- **중립성 유지**: 특정 학원 광고성 내용 포함 금지
- **오입력 수정**: 잘못 입력된 데이터는 Supabase Table Editor에서 직접 수정 가능
