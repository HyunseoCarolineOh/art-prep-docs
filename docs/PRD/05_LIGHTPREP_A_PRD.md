## 1. Problem (왜 지금?)

### 문제 정의
미대 입시생은 레퍼런스를 수집할 때 "왜 이 이미지가 이 대학/전형에 맞는지" 판단 근거가 부족하다.
- Pinterest/Instagram: 이미지만 있고 맥락 없음
- 학원 자료: 산발적, 설명 불충분
- 결과: 무분별한 수집 → 실기 대비 효율 저하

### 사용자 가치
- 학생: 레퍼런스 저장 시 "이 이미지가 왜 중요한지" AI 리포트로 즉시 이해
- 차별화: 단순 이미지 북마크 → 학습 근거가 있는 레퍼런스 관리

### 비즈니스 가치
- ArtPrep만의 차별화 기능 확보
- 사업계획서 AI 기능 1단계 구현 완료

### 왜 지금?
- 다음주부터 실제 유저 데이터 업로드 시작
- 차별화 기능 없으면 타 플랫폼과 동일하게 보임

## 3. Features (기능 명세)

### 3.1. 리포트 생성 플로우

```
[사용자] 레퍼런스 이미지 업로드
    ↓
[AI] 이미지 분석 + 5블록 리포트 자동 생성
    ↓
[화면] 리포트 표시 (저장 버튼)
    ↓
[DB] 저장 클릭 시 artworks 테이블에 JSONB로 저장
```

### 3.2. 리포트 5블록 구조

| 블록 | 내용 | 데이터 출처 |
|------|------|-------------|
| **1. 출제 문제** | 해당 대학/전형의 대표 실기 주제 (예: "20XX년 OO대 실기 문제: 인물과 사물의 조화") | AI 생성 또는 관리자 입력 |
| **2. 문제 분석** | 출제 의도 및 평가 포인트 (예: "구도력·명암 대비 평가") | AI 생성 |
| **3. 작품 의도** | 이 레퍼런스가 보여주는 기법·구도·색감 (예: "역광 활용한 극적 분위기") | AI 이미지 분석 |
| **4. 평가 기준** | 대학별 채점 기준과 연계 (예: "창의성 30%, 기초 실력 40%") | AI 생성 또는 관리자 입력 |
| **5. 유사 합격작** | 비슷한 접근법의 합격작 1~2개 (텍스트 설명만, 저작권 이슈로 이미지 미포함) | AI 생성 또는 관리자 입력 |

**데이터 우선순위**:
1. AI 자동 생성 (Gemini Flash 2.0)
2. 관리자 수동 편집 (B안에서 구현)

### 3.3. UI/UX

**업로드 화면**:
- 기존 작품 업로드 화면 재사용
- "AI 리포트 생성" 버튼 추가

**리포트 화면**:
- 상단: 이미지 미리보기
- 하단: 5블록 카드 형태 순차 표시 (접기/펼치기 가능)
- 각 블록: 제목 + 2~3문장 설명
- 하단 액션: [저장하기] [다시 생성하기]

**저장 후**:
- "저장되었습니다" 토스트
- 내 작품 목록에서 리포트 다시 보기 가능

## 4. Tech Stack

### 4.1. AI API
- **선택**: Google Gemini Flash 2.0 (무료 tier)
- **이유**:
  - 무료 한도: 월 60건 (RPM 15)
  - 이미지 분석 + 텍스트 생성 동시 지원
  - 한국어 출력 품질 양호
- **대안**: Claude API (유료 전환 시)
- **Fallback**: API 실패 시 "일시적으로 리포트 생성 불가" 메시지 + 이미지만 저장 허용

### 4.2. DB 스키마

`artworks` 테이블에 컬럼 추가:

```sql
ALTER TABLE artworks
ADD COLUMN ai_report JSONB;
```

**JSONB 구조 예시**:

```json
{
  "generated_at": "2026-05-13T10:30:00Z",
  "model": "gemini-2.0-flash-exp",
  "blocks": [
    {
      "type": "exam_question",
      "title": "출제 문제",
      "content": "2025년 홍익대 실기 문제: 주어진 오브제를 활용한 자유 구성"
    },
    {
      "type": "analysis",
      "title": "문제 분석",
      "content": "이 문제는 공간 구성력과 창의적 발상을 평가합니다..."
    }
  ]
}
```

### 4.3. 프론트엔드
- Next.js API Route: `/api/ai/generate-report`
- 컴포넌트: `<ArtworkReportCard />`, `<ReportBlock />`

### 4.4. 백엔드 로직

```
POST /api/ai/generate-report
Body: { artwork_id, image_url }

1. Supabase Storage에서 이미지 가져오기
2. Gemini API 호출 (이미지 + 프롬프트)
3. 응답 파싱 → JSONB 구조로 변환
4. artworks.ai_report에 저장
5. 클라이언트에 응답 반환
```

## 5. Scope (구현 범위)

### 만드는 것 (A안)
- 레퍼런스 이미지 업로드 시 "AI 리포트 생성" 버튼
- Gemini API 연동 (이미지 → 5블록 텍스트)
- 리포트 5블록 화면 표시
- 리포트 저장 기능 (DB JSONB)
- 저장된 리포트 다시 보기
- 무료 API 한도 모니터링 (월 60건)

### 만들지 않는 것 (Scope Out)
- **B안 기능**: 촉진 질문 3개 + 학생 답변 저장 (A안 완성 후 2단계로)
- 관리자 리포트 편집 기능
- 리포트 공유/내보내기 (저작권·URL 공유)
- 리포트 버전 관리
- 다국어 지원 (한국어만)
- 리포트 템플릿 커스터마이징
- 유사 합격작 이미지 (저작권 — 텍스트만)

**이유**: A안 먼저 완성 → 유저 테스트 → 피드백 반영 후 B안 진행. 관리자 기능은 추후 필요성 검증 후 추가.

## 6. Edge Cases

| 케이스 | 처리 방법 |
|--------|-----------|
| **이미지 분석 실패** (너무 작음·손상·예술 작품 아님) | Gemini 에러 응답 시 "이미지를 분석할 수 없습니다. 다른 이미지를 선택해주세요" 토스트 + 재시도 버튼 |
| **이미지가 실기와 무관** (예: 풍경 사진) | AI가 "실기 레퍼런스로 부적합" 메시지 생성 |
| **API 무료 한도 초과** (월 60건) | "현재 많은 요청으로 인해 잠시 후 다시 시도해주세요" + 관리자 알림 (Slack/이메일) + 유료 전환 검토 |
| **부적절한 이미지** (음란·폭력) | Gemini Safety Settings (`BLOCK_MEDIUM_AND_ABOVE`) → 차단 시 "부적절한 콘텐츠 감지" + 저장 안 됨 |
| **리포트 생성 지연** (10초+) | 로딩 스피너 (최대 30초 timeout) → 초과 시 "생성 실패" 메시지. 백그라운드 큐는 V2에서 |
| **대학/전형 정보 없음** | "전형 선택 필수" 안내 |
| **중복 생성 방지** | 생성 중 버튼 비활성화. 기존 리포트 있으면 "이미 리포트가 있습니다. 다시 생성?" 확인창 |

## 9. Open Questions (미결 사항)

### Q1. 프롬프트 구체화 — "출제 문제" 생성 방식
- 옵션 A: 이미지만으로 추측 (예: "이 작품은 공간 구성 문제로 보입니다")
- 옵션 B: 관리자가 사전 입력한 "학교별 기출 문제" DB 참조
- **→ 일단 A로 진행, 추후 B 추가**

### Q2. 리포트 재생성 정책
- 한 번 생성 후 다시 생성 가능?
- **→ 가능. "다시 생성하기" 버튼 제공 (단, API 한도 소진)**

### Q3. 기존 작품 처리
- 이미 업로드된 작품도 리포트 생성 가능?
- **→ 가능. 작품 상세 페이지에서 "리포트 생성" 버튼 추가**

## Appendix A. Prompt Template

```
당신은 미대 입시 전문 컨설턴트입니다.
학생이 업로드한 레퍼런스 이미지를 분석하여, 다음 5가지 블록으로 구성된 리포트를 작성해주세요.

[이미지: {image_url}]

출력 형식 (JSON):
{
  "blocks": [
    {
      "type": "exam_question",
      "title": "출제 문제",
      "content": "이 작품이 적용될 수 있는 대학 실기 문제를 추정하여 작성하세요. (예: 2025년 홍익대 실기 문제: ...)"
    },
    {
      "type": "analysis",
      "title": "문제 분석",
      "content": "이 문제가 평가하는 능력(구성력, 표현력, 창의성 등)을 2~3문장으로 설명하세요."
    },
    {
      "type": "intention",
      "title": "작품 의도",
      "content": "이 레퍼런스 이미지가 보여주는 기법, 구도, 색감 등을 분석하여 2~3문장으로 설명하세요."
    },
    {
      "type": "criteria",
      "title": "평가 기준",
      "content": "주요 미대(홍익대, 서울대 등)의 평가 기준 중 이 작품이 충족하는 요소를 설명하세요."
    },
    {
      "type": "similar_works",
      "title": "유사 합격작",
      "content": "이 스타일과 유사한 합격작 사례를 언급하며, 공통점을 설명하세요. (실제 데이터가 없으면 일반적인 경향 설명)"
    }
  ]
}

주의사항:
- 각 content는 2~3문장으로 간결하게 작성
- 미대 입시생이 이해할 수 있는 쉬운 용어 사용
- 구체적인 예시 포함 (예: "상단의 빈 공간이...", "색감 대비를 통해...")
```
