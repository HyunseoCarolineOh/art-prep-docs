사용자 행동 데이터 09 / 15

추천 모델을 학습시킬 두 개의 행동 데이터

사용자가 "좋아한 작품"과 "탐색한 흐름"이 모두 기록된다. 이 두 데이터로 개인화 모델을 만들 수 있다.

saved_items

사용자가 작품을 저장(♡)할 때마다 한 행이 쌓인다
1
학생 A가 작품 카드의 ♡ 클릭
2
{ user_id, artwork_id, created_at } 한 행이 INSERT
3
UNIQUE 제약으로 중복 저장 불가 — 항상 1:1 매핑
활용 ▸ 사용자별 선호 학교·태그·구도 분포 추출 → 협업 필터링 / 콘텐츠 기반 추천 학습

curation_sessions

사용자가 큐레이션 화면을 열 때마다 한 행이 쌓인다
1
학생이 필터 칩(학교·전형·태그) 선택
2
filter_params JSONB로 어떤 조건을 걸었는지 저장
3
result_ids에 보여준 작품 ID 배열 저장
활용 ▸ 노출 vs 저장 전환율 측정, 검색 의도 클러스터링, 추천 결과 품질 검증
두 테이블 모두 RLS(사용자별 행 단위 접근 제어) 적용 — 사용자는 본인 데이터만 조회 가능. BOAZ 팀에는 별도 읽기 전용 계정 발급 예정.

out: 의도 텍스트
2
학교 성향

in: 1단계+학교
out: 매칭 점수
3
조형 요소
구도·명암·색채·디테일을 항목별로 분해해 분석
in: image
out: 항목별 점수
4
평가 기준
대학별 채점 기준에 비추어 강점·약점 도출

out: 강·약점 리스트
5
합격 가능성

out: 확률 + 코칭
