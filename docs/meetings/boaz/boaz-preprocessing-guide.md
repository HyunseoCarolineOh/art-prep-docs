## 0. 작업 배경

- 전처리로 보존하는 작품들은 AI 학습을 시켜 feature embedding 하는 데 쓰임
- 모델은 vision encoder로 작품을 처리함
- 따라서 작품 외 잡음이 포함되지 않도록 작품만 깔끔하게 크롭하는 작업이 필요

## Task 1. 합격작 크롭 (재크롭)

`크롭된 사진` 폴더를 메인으로 작업. 크롭이 제대로 안 된 사진은 `원본 사진` 폴더에서 원본을 찾아 재크롭 후 **기존 파일에 덮어쓰기**.

**파일명 규칙**: 잘못 크롭된 파일이 `abcde(1)`이면 원본 폴더에서 `abcde`를 찾아 재크롭 → `abcde(1)`로 저장 (덮어쓰기)

**"크롭이 제대로 안 된 사진" 기준**

| 케이스 | 처리 |
|--------|------|
| 합격작과 다른 그림이 함께 첨부 | 합격작만 따로 재크롭 |
| 글자가 합격작 내부에 첨부 | 글자 포함한 채 합격작 전체 크롭 |
| 글자가 합격작 외부에 첨부 | 글자 안 나오게 합격작만 크롭 |
| 합격작 일부만 크롭됨 (잘려 보임) | 원본 찾아 전체 재크롭 |
| 완성작 아닌 과정샷 크롭 | 보통 과정샷 3컷 + 완성작 1컷 구성, 완성작(우측 하단)을 크롭 |
| 누가 봐도 합격작 아님 (학원 로고·이모지·아이콘·미완성·너무 대충 그린 그림) | (Task 3 기준으로 처리) |

## Task 2. 중복 크롭 사진 제거

동일 작품에 크롭이 2번 이상 진행된 경우 하나의 파일로 통일. (예: `abcde(1)`, `abcde(1)(1)`, `abcde(1)(2)` 같은 중복)

## Task 3. 미완성작 제거

| 기준 | 처리 |
|------|------|
| 채색 안 됨 + 완성도 낮음 | 제거 |
| 채색 안 됐어도 스케치 완성도 높음 또는 소묘작 | 보존 |
| 완성도 판별 애매 | 검수자(전공자) 판단: vision encoding에 충분한가 |
| 채색이 되다 만 경우 | 특징 학습에 충분한 완성도면 보존, 아니면 폐기 |

**주의**: 합격작이 아닌 것 같아도 `크롭된 사진` 폴더에 있으면 임의 삭제 금지. Task 1~3 동일 적용하고, 합격작 여부는 모델링 담당자가 추후 처리.

## Task 4. 특성 문구 라벨링

(원문에는 "Task 3"으로 번호 중복 표기됨)

- 각 작품에 한 줄 문구 작성
- `.xlsx` 파일, 파일명 `특성 문구 라벨링`, Artprep팀 폴더(공유 폴더 루트)에 업로드
- 양식: `작품명 | 문구` 2컬럼 (작품명 = 크롭 파일명, 예: `abcde(1)`)

### 작성 규칙 (2026-06-11 킥오프 확정 — 이 규칙이 기준, 이전 파일럿 형식은 폐기)

**원칙**
- 눈에 보이는 것만 객관적으로 기술. 주관적 감상·가치판단·감정 표현 금지
- 알아보기 쉬운 단어만 사용
- 한 작품당 한 문장, 끝은 작품 종류로 마무리 (예: 소묘 작품, 수채 작품, 마커 발상과표현 작품)

**반드시 포함할 분석 요소**
1. 무엇이 있는가: 화면에 등장하는 주요 오브제와 색·형태·질감
2. 구도와 레이아웃: 주요 대상의 위치(좌측/우측/중앙/상단/하단), 시점(정면·투시·부감·앙시), 배치 축(수직·수평·사선·삼각), 면 분할·시선 흐름
3. 화면 운영: 여백과 밀집의 분포, 원근·깊이감, 명암·색 대비가 화면을 나누는 방식

**작성 순서**: ①오브제와 색·질감 → ②구도·시점·배치 축 → ③작품 종류로 마무리

**예시**
- "화면 좌측 하단에 인물 두 명을 세로로 배치하고, 우측에는 유리에 비친 실내를 노란 톤으로 펼쳐 좌우를 명암으로 가른 채색 인물 작품"
- "중앙 나무판을 수평축으로 두고 좌상단에서 우하단으로 흐르는 물줄기를 사선 구도로 연결한 연필 소묘 작품"
