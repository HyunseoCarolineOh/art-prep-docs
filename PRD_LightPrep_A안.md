# PRD: LightPrep A안 (AI 리포트 생성)

**작성일**: 2026-05-13  
**작성자**: PM  
**목표 출시일**: 2026-05-20 (다음 수업 시연)

---

## 1. 왜 지금?

### 문제 정의
미대 입시생은 레퍼런스를 수집할 때 "왜 이 이미지가 이 대학/전형에 맞는지" 판단 근거가 부족하다.
- 핀터레스트/인스타: 이미지만 있고 맥락 없음
- 학원 자료: 산발적, 설명 불충분
- 결과: 무분별한 수집 → 실기 대비 효율 저하

### 사용자 가치
- 학생: 레퍼런스 저장 시 "이 이미지가 왜 중요한지" AI 리포트로 즉시 이해
- 차별화: 단순 이미지 북마크 → 학습 근거가 있는 레퍼런스 관리

---

## 2. Success Metric (성공 기준)

| 지표 | 목표 |
|------|------|
| **리포트 생성 성공률** | 95% 이상 (API 호출 실패 5% 이하) |
| **사용자 저장 비율** | 리포트 생성 후 50% 이상이 레퍼런스를 실제 저장 |
| **리포트 읽기 완독률** | 생성된 리포트 중 70% 이상 끝까지 스크롤 |
| **API 비용** | 무료 tier 내 (월 60건 이하, Gemini Flash 기준) |

**검증 방법**: 다음주 실제 유저 2~3명에게 테스트 → 리포트 유용성 인터뷰

---

## 3. Scope (구현 범위)

### ✅ 이번에 만드는 것 (A안)

#### 3.1. 리포트 생성 로직
- **입력**: 레퍼런스 이미지 URL + 대학/전형 정보 (예: 홍익대 회화과 정시)
- **AI 생성 블록** (5개):
  1. **출제 문제**: 해당 대학/전형의 대표 실기 주제 (예: "인물과 사물의 조화")
  2. **문제 분석**: 출제 의도 및 평가 포인트 (예: "구도력, 명암 대비")
  3. **작품 의도**: 이 레퍼런스가 보여주는 기법/컨셉 (예: "역광 활용한 극적 분위기")
  4. **평가 기준**: 대학별 채점 기준과 연계 (예: "창의성 30%, 기초 실력 40%")
  5. **유사 합격작 사례**: 비슷한 접근법의 합격작 1~2개 (텍스트 설명만, 이미지 없음)
- **출력**: 마크다운 형식 텍스트 (DB 저장 + 화면 렌더링)

#### 3.2. UI/UX
- **트리거 지점**: 레퍼런스 업로드 후 "AI 리포트 생성" 버튼
- **로딩 상태**: "분석 중..." 스피너 (예상 시간: 3~5초)
- **리포트 표시**: 
  - 화면 상단에 이미지 미리보기
  - 하단에 5블록 리포트 (접기/펼치기 가능)
  - "저장하기" 버튼 (리포트 + 이미지 함께 저장)

#### 3.3. 데이터 저장
```sql
-- references 테이블에 ai_report 컬럼 추가 (JSONB)
{
  "problem": "출제 문제 텍스트",
  "analysis": "문제 분석 텍스트",
  "intent": "작품 의도 텍스트",
  "criteria": "평가 기준 텍스트",
  "similar_cases": "유사 합격작 텍스트"
}
```

#### 3.4. API 선택
- **무료 대안**: Google Gemini Flash 2.0 (월 60건 무료)
- **fallback**: API 실패 시 "일시적으로 리포트 생성 불가" 메시지 + 이미지만 저장 허용

---

### ❌ 이번에 만들지 않는 것 (Scope Out)

- **B안 기능**: 촉진 질문 (A안 완성 후 2단계로 연기)
- **관리자 수정 기능**: AI 생성 리포트 수동 편집 (향후)
- **유사 합격작 이미지**: 텍스트 설명만 (저작권 이슈)
- **다국어 지원**: 한국어만 (추후 영어 추가 검토)
- **리포트 공유**: 저장만 가능, URL 공유는 미지원

## 4. 기술 스택

| 항목 | 선택 |
|------|------|
| **AI API** | Google Gemini Flash 2.0 (무료) |
| **Vision API** | Gemini 멀티모달 (이미지 분석) |
| **프롬프트 구조** | 시스템 메시지에 대학/전형 맥락 주입 |
| **DB 스키마** | references.ai_report (JSONB) |
| **Frontend** | React (기존 artprep 코드베이스) |

---

## 5. Edge Cases (예외 상황 처리)

| 케이스 | 처리 방법 |
|--------|-----------|
| **이미지가 실기와 무관** (예: 풍경 사진) | AI가 "실기 레퍼런스로 부적합" 메시지 생성 |
| **API 응답 지연** (10초 이상) | 타임아웃 후 재시도 1회, 실패 시 저장만 허용 |
| **대학/전형 정보 없음** | "전형 선택 필수" 안내 메시지 |
| **월 무료 한도 초과** | "이번 달 리포트 생성 횟수 초과" + 유료 전환 안내 |

## 6. 구현 계획

### Phase 1 (이번 주)
1. Gemini API 연동 테스트 (간단한 프롬프트)
2. 리포트 생성 로직 구현 (5블록 구조)
3. UI 컴포넌트 (버튼 + 리포트 표시)
4. DB 스키마 추가 (ai_report 컬럼)

### Phase 2 (다음 주)
1. 실제 유저 테스트 (2~3명)
2. 프롬프트 튜닝 (리포트 품질 개선)
3. 버그 수정 및 UX 개선

## 7. 리스크 및 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| **Gemini 무료 한도 부족** | 높음 | 사용량 모니터링 + 유료 전환 기준 설정 |

| **저작권 이슈** (유사 합격작) | 낮음 | 텍스트 설명만 제공 (이미지 미포함) |
