# LightPrep A안 PRD (Product Requirements Document)

**작성일**: 2026-05-13  
**버전**: 1.0  
**담당**: PM

---

## 1. Problem (문제 정의)

**현재 상황**:
- 미대 입시생들이 Pinterest, Instagram에서 레퍼런스 이미지를 수집하지만, **"왜 이 이미지가 내 지망 학교에 맞는지"** 분석할 방법이 없음
- 수집만 하고 끝나는 경험 → artprep의 차별점이 필요

**사용자 불편**:
- 레퍼런스를 보고도 "이게 왜 좋은지" 스스로 판단하기 어려움
- 학원 선생님께 물어봐야 하는데 즉시 피드백 받기 힘듦

**왜 지금?**
- 다음주부터 실제 유저 데이터 업로드 시작
- 차별화 기능 없으면 타 플랫폼과 동일하게 보임

---

## 2. Goal (목표)

**사용자 가치**:
- 레퍼런스 업로드 → AI가 즉시 5블록 리포트 생성 → "왜 이 이미지가 이 학교에 맞는지" 이해

**비즈니스 가치**:
- artprep만의 차별화 기능 확보
- 사업계획서 AI 기능 1단계 구현 완료

---

## 3. Success Metrics (성공 지표)

### 정량 지표
- **리포트 생성 후 저장 비율 50% 이상**
  - 생성된 리포트 중 절반 이상을 사용자가 실제로 저장/활용
- **다음주 유저 테스트**: 2~3명 피드백 수집

### 정성 지표
- "이 분석이 도움 되었나요?" 사용자 인터뷰
- "다른 레퍼런스도 분석하고 싶은가요?" 재사용 의향

---

## 4. Features (기능 명세)

### 4.1. 리포트 생성 플로우

```
[사용자] 레퍼런스 이미지 업로드
    ↓
[AI] 이미지 분석 + 5블록 리포트 자동 생성
    ↓
[화면] 리포트 표시 (저장 버튼)
    ↓
[DB] 저장 클릭 시 artworks 테이블에 JSONB로 저장
```

### 4.2. 리포트 5블록 구조

| 블록 | 내용 | 데이터 출처 |
|------|------|-------------|
| **1. 출제 문제** | "20XX년 OO대 실기 문제: ..." | AI 생성 또는 관리자 입력 |
| **2. 문제 분석** | "이 문제는 [구성력/표현력]을 평가합니다" | AI 생성 |
| **3. 작품 의도** | "이 레퍼런스는 [공간 분할/색감 대비] 등을 보여줍니다" | AI 이미지 분석 |
| **4. 평가 기준** | "OO대는 [발상/완성도/창의성] 중심" | AI 생성 또는 관리자 입력 |
| **5. 유사 합격작** | "작년 합격작 A, B와 비교..." | AI 생성 또는 관리자 입력 |

**데이터 우선순위**:
- 1차: AI 자동 생성 (Gemini Flash)
- 2차: 관리자가 수동 편집 (추후 B안에서 구현)

### 4.3. UI/UX

**업로드 화면**:
- 기존 작품 업로드 화면 재사용
- "AI 리포트 생성" 버튼 추가

**리포트 화면**:
- 5블록 카드 형태로 순차 표시
- 각 블록: 제목 + 2~3문장 설명
- 하단: [저장하기] [다시 생성하기] 버튼

**저장 후**:
- "저장되었습니다" 토스트 메시지
- 내 작품 목록에서 리포트 다시 보기 가능

## 5. Tech Stack (기술 스택)

### 5.1. AI API
- **선택**: Google Gemini Flash 2.0 (무료 tier)
- **이유**:
  - 무료 한도: 월 60건 (RPM 15)
  - 이미지 분석 + 텍스트 생성 동시 지원
  - 한국어 출력 품질 양호
- **대안**: Claude API (유료 전환 시)

### 5.2. DB 스키마

**artworks 테이블에 컬럼 추가**:

```sql
ALTER TABLE artworks
ADD COLUMN ai_report JSONB;
```

**JSONB 구조 예시**:

```json
{
  "generated_at": "2026-05-13T10:30:00Z",
  "model": "gemini-2.0-flash-exp",
  "blocks": [
    {
      "type": "exam_question",
      "title": "출제 문제",
      "content": "2025년 홍익대 실기 문제: 주어진 오브제를 활용한 자유 구성"
    },
    {
      "type": "analysis",
      "title": "문제 분석",
      "content": "이 문제는 공간 구성력과 창의적 발상을 평가합니다..."
    },
    // ... 나머지 3블록
  ]
}
```

### 5.3. 프론트엔드
- Next.js API Route: `/api/ai/generate-report`
- 컴포넌트: `<ArtworkReportCard />`, `<ReportBlock />`

### 5.4. 백엔드 로직

```
POST /api/ai/generate-report
Body: { artwork_id, image_url }

1. Supabase Storage에서 이미지 가져오기
2. Gemini API 호출 (이미지 + 프롬프트)
3. 응답 파싱 → JSONB 구조로 변환
4. artworks.ai_report에 저장
5. 클라이언트에 응답 반환
```

## 6. Scope (구현 범위)

### ✅ 만드는 것
- [x] 레퍼런스 이미지 업로드 시 "AI 리포트 생성" 버튼
- [x] Gemini API 연동 (이미지 → 5블록 텍스트)
- [x] 리포트 5블록 화면 표시
- [x] 리포트 저장 기능 (DB JSONB)
- [x] 저장된 리포트 다시 보기
- [x] 무료 API 한도 모니터링 (월 60건)

### ❌ 안 만드는 것 (Out of Scope)
- [ ] B안 기능 (촉진 질문 3개 + 학생 답변 저장)
- [ ] 관리자 리포트 편집 기능
- [ ] 리포트 공유/내보내기
- [ ] 리포트 버전 관리
- [ ] 다국어 지원 (한국어만)
- [ ] 리포트 템플릿 커스터마이징

**이유**: 
- A안 먼저 완성 → 유저 테스트 → 피드백 반영 후 B안 진행
- 관리자 기능은 추후 필요성 검증 후 추가

## 7. Edge Cases (엣지케이스)

### 7.1. 이미지 분석 실패
**상황**: 이미지가 너무 작거나, 손상되었거나, 예술 작품이 아닌 경우
**처리**:
- Gemini API 에러 응답 시 → "이미지를 분석할 수 없습니다. 다른 이미지를 선택해주세요" 토스트
- 재시도 버튼 제공

### 7.2. API 무료 한도 초과
**상황**: 월 60건 초과 (RPM 15 제한)
**처리**:
- 초과 감지 → "현재 많은 요청으로 인해 잠시 후 다시 시도해주세요" 메시지
- 관리자 알림 (Slack/이메일)
- 유료 전환 검토 (다음 phase)

### 7.3. 부적절한 이미지
**상황**: 음란물, 폭력적 이미지 등
**처리**:
- Gemini Safety Settings 활용 (기본값: BLOCK_MEDIUM_AND_ABOVE)
- 차단 시 → "부적절한 콘텐츠가 감지되었습니다" 메시지
- 저장 안 됨

### 7.4. 리포트 생성 지연
**상황**: API 응답이 10초 이상 걸리는 경우
**처리**:
- 로딩 스피너 표시 (최대 30초 timeout)
- 30초 초과 시 → "생성에 실패했습니다. 다시 시도해주세요" 메시지
- 백그라운드 큐 방식은 V2에서 고려

### 7.5. 중복 생성 방지
**상황**: 사용자가 "생성" 버튼을 연타하는 경우
**처리**:
- 생성 중에는 버튼 비활성화
- 이미 리포트가 있으면 → "이미 리포트가 있습니다. 다시 생성하시겠습니까?" 확인창

## 9. Open Questions (미결 사항)

### Q1. 프롬프트 구체화
- "출제 문제"를 어떻게 생성할까?
  - 옵션 A: 이미지만으로 추측 (예: "이 작품은 공간 구성 문제로 보입니다")
  - 옵션 B: 관리자가 사전 입력한 "학교별 기출 문제" DB 참조
  - **→ 일단 A로 진행, 추후 B 추가**

### Q2. 리포트 재생성 정책
- 한 번 생성 후 다시 생성 가능한가?
  - **→ 가능. "다시 생성하기" 버튼 제공 (단, API 한도 소진)**

### Q3. 리포트 없는 기존 작품 처리
- 이미 업로드된 작품들도 리포트 생성 가능하게?
  - **→ 가능. 작품 상세 페이지에서 "리포트 생성" 버튼 추가**

## Appendix: Prompt Template (프롬프트 템플릿)

```
당신은 미대 입시 전문 컨설턴트입니다.
학생이 업로드한 레퍼런스 이미지를 분석하여, 다음 5가지 블록으로 구성된 리포트를 작성해주세요.

[이미지: {image_url}]

출력 형식 (JSON):
{
  "blocks": [
    {
      "type": "exam_question",
      "title": "출제 문제",
      "content": "이 작품이 적용될 수 있는 대학 실기 문제를 추정하여 작성하세요. (예: 2025년 홍익대 실기 문제: ...)"
    },
    {
      "type": "analysis",
      "title": "문제 분석",
      "content": "이 문제가 평가하는 능력(구성력, 표현력, 창의성 등)을 2~3문장으로 설명하세요."
    },
    {
      "type": "intention",
      "title": "작품 의도",
      "content": "이 레퍼런스 이미지가 보여주는 기법, 구도, 색감 등을 분석하여 2~3문장으로 설명하세요."
    },
    {
      "type": "criteria",
      "title": "평가 기준",
      "content": "주요 미대(홍익대, 서울대 등)의 평가 기준 중 이 작품이 충족하는 요소를 설명하세요."
    },
    {
      "type": "similar_works",
      "title": "유사 합격작",
      "content": "이 스타일과 유사한 합격작 사례를 언급하며, 공통점을 설명하세요. (실제 데이터가 없으면 일반적인 경향 설명)"
    }
  ]
}

주의사항:
- 각 content는 2~3문장으로 간결하게 작성
- 미대 입시생이 이해할 수 있는 쉬운 용어 사용
- 구체적인 예시 포함 (예: "상단의 빈 공간이...", "색감 대비를 통해...")
```
